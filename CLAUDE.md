# Kagoshima Circular Hub - Claude Code 指示書

## プロジェクト概要

**サイト名**: 鹿児島サーキュラーエコノミー情報ポータル（Kagoshima Circular Hub）
**コンセプト**: 鹿児島の「点」を「円（サークル）」につなぐ実務直結型デジタル・プラットフォーム
**GitHubリポジトリ**: https://github.com/kentatanaka-kago/kagoshima-circular-hub
**目的**: 県内の行政・企業・支援者が持つ断片的な情報を、2026年度から本格化する法規制対応や実務に即した「動ける知見」へと構造化する

---

## 技術スタック

| コンポーネント | 技術 | 備考 |
|--------------|------|------|
| フロントエンド | Next.js (TypeScript) | App Router, Tailwind CSS |
| ホスティング | Vercel | GitHubと連携済み |

| AI要約エンジン | OpenAI API (GPT-4o) | 行政PDFの実務サマリー化 |

## 機能ロードマップ

### 第1フェーズ（即時実装）: 情報アグリゲーション
- 毎朝6:40の自動更新（県内43市町村・主要ニュースサイト巡回）
- 行政用語の翻訳機能（「誰が・いつまでに・いくら貰えるか」に要約）
- 支援施策カレンダー（環境省補助金〜地域支援事業を一元管理）

### 第2フェーズ（2026年6月報告対応）: 報告実務支援
- 「特定産業廃棄物処分業者」向け報告ガイド
- データ構造化の雛形配布（Excel→DB移行用標準テーブル設計）

### 第3フェーズ（機能拡張）: 脱炭素・CE可視化
- 簡易脱炭素計算機（環境省排出原単位DBと連携）
- CE拠点マップ（Google Maps APIまたはOSM）

## ターゲットユーザー

1. **静脈側中小企業（30〜50社）**: 国への報告義務・メーカーへの証明書発行に追われる実務担当者
2. **動脈側サプライヤー**: 大手からデータ開示を求められる中小製造業
3. **自治体担当者**: 地域内の資源循環率を可視化し施策のエビデンスを必要とする担当者

## 法的・技術的留意事項

- スクレイピングは自治体サイトの利用規約を遵守し、過度なアクセスを避ける
- 著作権法に配慮し「要約・出典リンク」の形式を徹底する
- 排出原単位などの計算ロジックは環境省等の公的出典（マスバランス方式等）を明記する

## 作業開始時の確認事項

1. `npm run dev` でローカル起動できるか確認
2. Supabaseプロジェクトの作成・接続設定
3. Vercelへの初回デプロイ（`vercel --prod`）
4. 環境変数（`.env.local`）の設定
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `OPENAI_API_KEY`

## 最初に実装すること

第1フェーズの「情報アグリゲーション」から着手する。
まずは以下のページ構成でスタート：

```
/                    # トップページ（最新ニュース一覧）
/news/[id]           # ニュース詳細
/calendar            # 支援施策カレンダー
/municipalities/[id] # 自治体別ページ（SEO用）
```
